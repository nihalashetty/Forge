# Forge — Fix & Feature Implementation Plan (2026-06-11)

Master checklist from the 2026-06-11 advanced analysis. Status legend:
`[ ]` pending · `[x]` done · `[~]` deliberately deferred (reason noted).

## A. Performance (~600ms/node diagnosis)

Measured: framework ~1.2ms/node, +SQLite checkpointing ~8ms/node. Real costs were
client construction (httpx client = ~470ms on this Windows box, OpenAIEmbeddings
ctor = ~1s) and duplicated work inside nodes.

- [x] A1. Shared `httpx.AsyncClient` singleton (`forge/util/http.py`); replace fresh-client
      construction at all 7 sites: tools/rest.py, tools/graphql.py, nodes/data.py (webhook_out),
      tools/builtin.py (web_fetch), auth_providers/resolver.py (×2), services/knowledge.py (url ingest).
      Per-request timeouts via `client.request(..., timeout=...)`.
- [x] A2. Cache embedder instances per (model, api-key) in `knowledge/embeddings.py` —
      avoids ~1s `OpenAIEmbeddings` ctor per call.
- [x] A3. Fix `_LCEmbedder` hardcoded `name="oai-1536"`/`dim=1536` — derive dim/name from the
      model (text-embedding-3-small=1536, -3-large=3072).
- [x] A4. `ChromaStore` client/collection cache (module-level, keyed by path+collection) —
      avoids re-init per query.
- [x] A5. Retrieval node embeds the query ONCE per execution and reuses the vector for both
      doc search and QA scoring (was: 2× embedder resolution + 2× embedding API calls).
      `KnowledgeService.search`/`top_qa` accept a precomputed embedding.
- [x] A6. `SecretStore.read_ref` no longer COMMITs on every read — `last_used_at` is updated
      at most once per 5 minutes.
- [x] A7. Preload heavy modules (langchain_openai, chromadb) in a background thread at startup —
      hides the 22s/9s first-import stall (Windows Defender) from the first run.
- [x] A8. Replace `debug` stream mode with the lighter `tasks` mode in runs.py
      (verified: tasks chunks carry `{name, error, result}`).
- [x] A9. `FORGE_RUN_DURABILITY` setting (default "async") passed to `graph.astream`.
- [~] A10. `DeltaChannel` for the messages channel — DEFERRED: beta API, reducer signature is
      sequence-based (incompatible with binary `add_messages` without a wrapper), and it changes
      checkpoint format. Revisit when langgraph marks it stable.

## B. Multi-intent / workflow engine

Diagnosis: classifier picks exactly ONE label; router takes ONE path → second half of a
two-part question is dropped. Router with empty default ends runs silently.

- [x] B1. `knowledge_search` builtin tool — agent-callable KB search (docs + Q&A, optional
      folder filter). The recommended fix for multi-intent support flows: one agent with
      tools instead of classifier→router. Registered in builtin.py, tool builder UI, and
      the assistant's `create_builtin_tool`.
- [x] B2. Multi-label classifier: `multi_label: true` config → structured output returns a
      list of labels written to `output_key` (list[str]); keyword fallback collects all matches.
- [x] B3. Parallel router: `multi: true` config → returns ALL matching case targets (LangGraph
      fans out in parallel within one superstep; verified). Specialists then converge on a
      downstream node (e.g. a synthesizer agent) before end.
- [x] B4. Validation warning: router with no `default` (and no else edge) can end the run
      silently with no answer.
- [x] B5. Schemas + canvas support: classifier.json `multi_label`, router.json `multi`,
      `nodeWrittenKeys` declares list[str] for multi-label classifiers, NODE_FIELDS forms.
- [~] B6. Orphan "Codex Router Regression" workflow (project 6688 doesn't exist) — automated
      deletion was blocked by the session safety policy (user data). It's invisible in the UI
      and harmless; to remove it yourself:
      `sqlite3 apps/api/.data/forge.db "delete from workflows where project_id='6688'"`.
- [~] B7. Rebuilding the user's "Test Workflow" as a single agent — NOT done automatically
      (user data). The assistant can now do it on request, and B1 provides the tool.

## C. Forge Assistant

- [x] C1. Conversation persistence: assistant runs on the app checkpointer with a per-panel
      `thread_id`; frontend sends only the new message. Todos/files/tool history survive turns.
- [x] C2. Real HITL: `interrupt_on={"delete_workflow": True}` + `/assistant/resume` endpoint +
      approve/reject UI in the panel. Destructive ops now pause for approval.
- [x] C3. Generic `create_custom_workflow` tool — takes full nodes/edges/state JSON, validates
      via WorkflowService.validate, returns field-pointer errors so the agent can self-correct,
      builds the canvas, saves idempotently by name.
- [x] C4. Platform introspection tools: `list_node_types`, `get_node_schema`,
      `list_middleware_types` — read live from the registry + JSON schemas (no prompt drift).
- [x] C5. Model selection: project config `assistant_model` honored; stronger default map
      (openai→gpt-5.4 per June-2026 docs, anthropic→claude-sonnet-4-6) with
      `ModelFallbackMiddleware` falling back to the project default model.
- [x] C6. Evaluator step: `evaluate_build` LLM-judge tool (structured PASS/FAIL + issues);
      system prompt mandates judge-after-test in the build loop.
- [x] C7. DeepAgents skills: `forge-platform` skill (SKILL.md) mounted read-only via
      CompositeBackend(FilesystemBackend) — deep Forge how-to knowledge with progressive
      disclosure instead of prompt bloat.
- [x] C8. Stream `todos` updates to the panel (plan visibility) + expandable tool-result chips
      (was truncated at 200 chars with no way to see more).
- [x] C9. Folder/kind-aware assistant tools: add_knowledge_text(folder=), add_qa_pair(kind=).
- [~] C10. JWT auth on assistant/all endpoints — DEFERRED: Phase-7 platform-wide auth (login,
      users, tenancy) is a separate project-scale change; tracked in ROADMAP Phase 0/7.

## D. Misc engine/platform fixes

- [x] D1. Pricing table: added gemini-2.5-flash, gpt-4.1, gpt-5.4, claude-haiku-4-5,
      claude-opus-4-8, text-embedding-3-large; unknown models log a warn-once instead of
      silently pricing $0.
- [x] D2. `request_signing` middleware was a NO-OP stub listed in the catalog — removed from
      MW_BUILDERS, middleware.json, and the frontend catalog (honesty fix).
- [x] D3. `guardrail_regex` on_match="block" now actually REPLACES the model reply
      (RemoveMessage + replacement) instead of appending a second message.
- [x] D4. `tool_retry.retry_on` string exception names now map to real exception types
      (timeout/http/value/key/runtime/connection), instead of being ignored.
- [x] D5. Playground thread reuse: `RunCreate.thread_id` — playground + canvas test panel keep
      one thread per session and send only the new message (checkpointer holds history; no more
      full-transcript replay per turn). Reset on workflow switch / clear.
- [x] D6. `_PROVIDER_DEFAULT_MODEL` openai default modernized to gpt-4.1-mini.
- [~] D7. `tenant_budget` reading accumulated cost from traces — DEFERRED (needs design:
      per-run token budget works today; cross-run budget needs trace aggregation hooks).
- [x] D8. **Deleting a KB source left its vectors in Chroma** (deleted docs kept appearing in
      search). Root cause: `delete_source` called a bare `ChromaStore()` (default `forge_kb`
      collection) while vectors live in the dim-keyed `forge_kb_<dim>` collection — so the
      delete hit an empty collection. Fixed: resolve the project embedder and delete from its
      collection + sweep all known dim collections. Also purged the existing orphans from the
      live dev DB (5 sources incl. Westcon Comstor/GDPR docs) and verified add→search→delete→
      search round-trip removes the chunk.

- [x] E4. Assistant + API surface updated for folders/kinds (see C9).
